Identity & Age Verification

Every user on Fansbook must be 18+. Currently this relies on a self-declaration popup. This feature replaces that with document-based AI verification — ensuring legal compliance and protecting the platform.

How It Works

  1. User uploads a government-issued ID (passport, driving licence, national ID)
  2. Onfido / Veriff AI reads and validates the document in under 3 seconds
  3. Facial liveness check confirms the person holding the ID is real and present
  4. Result (Approved / Rejected / Manual Review) is sent to Fansbook backend
  5. Admin dashboard shows verification status per user; manual review queue for edge cases

What Gets Built

✓
ID Upload Flow — Secure upload screen in the user onboarding journey
✓
Liveness Detection — Selfie + ID face match with anti-spoofing
✓
Backend Webhook — Receive and store verification results in the database
✓
Admin Review Panel — View, approve, or reject flagged verifications
✓
Status Badges — Verified / Unverified indicators across user profiles
✓
Retry Logic — Users can re-submit if initial attempt fails

Technology

Onfido API Veriff (alternative) Node.js Webhook Handler Prisma DB React Upload UI

AI Content Moderation

Every image and video uploaded to Fansbook is automatically scanned before it goes live. This protects the platform from illegal content, CSAM, extreme violence, and policy violations — without requiring a full-time moderation team.

How It Works

  1. Creator uploads image or video
  2. Before storing, our backend sends the file to AWS Rekognition / Google Vision AI
  3. AI returns a confidence score across categories: explicit, suggestive, violence, etc.
  4. Content above threshold is auto-blocked; borderline content is flagged for human review
  5. Admin sees a moderation queue with AI scores, can approve or remove flagged content
  6. Creator is notified if their content is removed with a reason

Fraud & Chargeback Prevention

Chargebacks and payment fraud are among the biggest revenue leaks on creator platforms. This feature uses AI to score every transaction in real time and block suspicious activity before it becomes a dispute.

How It Works

  1. Every payment on the platform passes through Stripe Radar (built into Stripe)
  2. Radar’s ML model — trained on billions of global transactions — scores the risk
  3. High-risk transactions are blocked or sent to 3D Secure challenge
  4. Custom rules are added on top: velocity checks, geo-anomalies, new account spending limits
  5. Admin dashboard shows fraud metrics, blocked payments, and dispute rates
  6. Optional: Sift integration for deeper behavioural fraud signals (account takeover, promotion abuse)

What Gets Built

✓
Stripe Radar Activation — Enable and configure built-in fraud ML
✓
Custom Rule Set — Platform-specific rules (new accounts, high-value purchases, VPN detection)
✓
Admin Fraud Dashboard — Blocked transactions, risk scores, dispute tracking
✓
Velocity Limits — Rate-limit suspicious spending patterns
✓
Dispute Alerts — Real-time notifications when chargebacks are filed
✓
Reporting — Monthly fraud summary for the client

Technology

Stripe Radar Stripe Webhooks Custom Rule Engine Node.js React Admin Dashboard

Creator Chat & Engagement AI

Creators on Fansbook often manage hundreds of fan conversations simultaneously. This feature gives creators an AI assistant inside their message inbox — suggesting smart replies, drafting messages in their tone, and helping them stay engaged with fans without burning out.

How It Works

  1. Creator opens a fan conversation in their inbox
  2. An AI suggestion bar appears below the message input with 2–3 smart reply options
  3. Creator can tap a suggestion to use it as-is, or edit before sending
  4. Over time, the AI learns the creator’s tone and style (based on their message history)
  5. Creators can also type a rough idea and ask AI to polish it into a full message
  6. All AI usage is optional — creators can ignore suggestions entirely

Creator Upsell Advisor

Most creators leave money on the table without realising it. This feature deploys an AI advisor inside the creator dashboard that analyses performance data and delivers personalised, actionable recommendations to help creators grow their revenue.

How It Works

  1. The system collects creator data: subscriber count, tier pricing, post frequency, engagement rate, earnings history
  2. This data is sent to Claude API with a structured prompt built for creator monetisation advice
  3. AI generates 3–5 prioritised recommendations specific to that creator
  4. Insights are displayed as a card on the creator dashboard, refreshed weekly
  5. Examples: “Your top 12 fans haven’t had a new post in 8 days — post today to reduce churn”, “Your $9.99 tier has 80 subscribers — consider a premium $24.99 tier”
  6. Admin can view aggregate AI insights across all creators

AI Support Chatbot

Support requests consume significant time and cost. This feature deploys a 24/7 AI chatbot that handles the majority of common questions instantly — with seamless escalation to a human agent when needed.

How It Works

  1. A chat widget appears on all pages for logged-in users (bottom-right corner)
  2. User types a question; the AI searches a knowledge base of FAQs, policies, and platform guides
  3. Claude AI generates a helpful, accurate response in plain language
  4. If the AI cannot confidently answer, it escalates to a human support agent with full conversation history
  5. Admin can add/edit knowledge base articles to keep the AI up to date
  6. All conversations are logged; admin can review and improve AI responses over time
